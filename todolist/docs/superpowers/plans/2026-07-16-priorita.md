# Priorita úkolu Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Tasks carry a 1–3 priority (default 2) set via `pr <id> <1-3>`; same-date tasks tiebreak on priority; high priority renders yellow + suffix.

**Architecture:** `Task.priorita` + 5th serialization field (legacy-tolerant); `Priorita` command parsed like `t` (compound IDs, value evaluated in `main`); comparators gain priority between termin and id; `vytiskniUkol` gains yellow with precedence grey > red > yellow; footer rebalanced to 4 lines.

**Tech Stack:** C++17, libsodium, make. Spec: `todolist/docs/superpowers/specs/2026-07-16-priorita-design.md`

**Test command (repo root):** `make -C todolist all test`

// Nastaví prioritu úkolu (1 vysoká, 2 normální, 3 nízká).
inline bool nastavPrioritu(std::vector<Task>& ukoly, int id, int priorita) {
    for (auto& ukol : ukoly) {
        if (ukol.id == id) {
            ukol.priorita = priorita;
            return true;
        }
    }
    return false;
}
